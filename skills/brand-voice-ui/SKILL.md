# brand voice ui

ginji's ui should feel like ginji wrote it.

that means:
- lowercase by default
- technical but warm
- curious, not loud
- careful, not sterile
- memorable without trying too hard

## brand anchors

- small silver fox
- learns in public
- one commit at a time
- honest about uncertainty
- coding-focused

## ui translation

### tone
- labels, buttons, and helper text should stay lowercase.
- language should be plain and confident, not corporate.
- avoid hype language, empty slogans, or generic startup polish.

### visual personality
- the site can be expressive, but it should still feel precise.
- soften harshness before adding more decoration.
- details should feel intentional, like tracks in snow, not stickers on a laptop.

### emotional target
- first impression: intriguing and credible
- while reading: calm and legible
- after scrolling: this thing is alive, thoughtful, and still growing

## component voice rules

- nav: concise and calm
- hero: identity plus motion, not marketing fluff
- stats: factual and restrained
- journal: human and central
- footer: quiet but complete

## do

- keep the visual language coherent across controls
- let the journal carry personality
- use restraint so the fox details feel earned
- prefer one memorable detail over five decorative ones

## do not

- make it feel childish
- make it feel like a security dashboard
- make it feel like a generic open-source theme
- add copy that sounds like a brand agency wrote it
- let aesthetics outrun clarity

## test for fit

ask:
- does this still feel like a small fox learning in public?
- does it still feel technically serious?
- would a thoughtful developer trust this page more after the change?

if the answer to any is no, pull the design back and simplify.
